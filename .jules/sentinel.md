## 2025-02-14 - Overly Permissive CORS on Vercel Serverless Proxy
**Vulnerability:** The Vercel serverless function `api/get-style.js` used `Access-Control-Allow-Origin: *`. This endpoint acts as a proxy for the authenticated MapTiler API, embedding the private `MAP_TILER` key. By setting a wildcard CORS policy, any third-party domain could query this endpoint directly from their users' browsers.
**Learning:** Even though serverless functions are meant to be consumed by specific frontends (like Vercel deployments), the default setup does not enforce origins. Developers often use `*` to quickly solve CORS errors during local development without realizing it turns their API into an open proxy, allowing malicious or third-party sites to drain API quotas at the developer's expense.
**Prevention:** Always use strict CORS allowlists for serverless endpoints that embed or proxy authenticated APIs. If local testing is needed, explicitly include `http://localhost:PORT` in the allowlist rather than resorting to `*`.
