## 2024-05-24 - Overly Permissive CORS Configuration
**Vulnerability:** The API endpoint `api/get-style.js` used a wildcard `*` for the `Access-Control-Allow-Origin` header, allowing any origin to request resources. This exposed the endpoint to potential Cross-Origin Resource Sharing (CORS) abuse and data leakage.
**Learning:** The wildcard was likely used for convenience during development, but the endpoint handles sensitive credentials indirectly (MapTiler API keys via environment variables). While the key itself is not leaked to the frontend, unrestricted access allows arbitrary domains to use this proxy to fetch map tiles, consuming the application's API quota.
**Prevention:** Implement strict CORS policies. Only allow specific trusted origins (like the application's production domain and local development URLs). In this case, `https://2gbxenvironment-2.vercel.app`, `http://localhost:8080`, and `http://127.0.0.1:8080`. Send the `Vary: Origin` header when dynamically setting the origin.
